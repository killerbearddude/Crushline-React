# Game Design Source

## Game Identity

Crushline is a production-graph game about building and stabilizing industrial systems.

The graph is the game.

The player connects machines through typed ports to transform resources, route byproducts, manage waste, stabilize power, and complete objectives.

## Core Design Pillars

1. Player-authored production graphs.
2. Functional progression over fixed recipe completion.
3. Expert-pack depth without expert-pack patchwork.
4. Plastic, data-driven production content.
5. Small playable slices before large content.
6. Graph-visible upgrades.
7. Waste as pressure and opportunity.
8. Victory as system behavior, not just item possession.

## Long-Term Endgame

Closed-Line Certification.

The player wins by assembling a Closed-Line Seed and maintaining certification conditions for a defined simulation period.

Potential certification conditions:

- power margin;
- waste management;
- water security;
- gas stability;
- repair autonomy;
- raw input dependency;
- final module assembly.

## Functional Victory Slots

Possible final module slots:

- Certified Structure Module;
- Certified Power Module;
- Certified Chemistry Module;
- Certified Atmosphere Module;
- Certified Recovery Module;
- Certified Control Module;
- Certified Water Module;
- Certified Repair Module.

Each slot may be satisfied by multiple valid routes.

## Design Rule

Do not force one prescribed recipe tree when a functional proof would be better.
